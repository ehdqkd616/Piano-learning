## 1. 시장 조사 및 경쟁 제품 분석

### 1.1 주요 경쟁 제품 비교

피아노 교육 소프트웨어 시장은 크게 MIDI 중심 제품과 마이크 인식 중심 제품, 그리고 하이브리드 방식으로 구분됩니다.

| 제품명 | 입력 방식 | 피드백 방식 | 악보 표시 | 게이미피케이션 | 플랫폼 |
|---|---|---|---|---|---|
| **Synthesia** | MIDI 전용 | 폴링 노트 + 색상 | 선택적 악보 | 점수, 온라인 랭킹 | Windows / Mac / iOS / Android |
| **Flowkey** | MIDI + 마이크 | 실시간 색상 피드백 | 인터랙티브 악보 | 진도 추적 | Web / iOS / Android |
| **Simply Piano** | MIDI + 마이크 | 실시간 + 대기 모드 | 악보 중심 | 레벨업, 배지 | iOS / Android / Web |
| **Piano Marvel** | MIDI 전용 | 점수 + 상세 분석 | 악보 중심 | SASR 시험, 트로피 | Windows / Mac / Web |
| **Playground Sessions** | MIDI 전용 | 노트 히트/미스 | 악보 + 영상 | 점수, 챌린지 | Windows / Mac / iOS / Android |

### 1.2 입력 방식별 기술 분석

#### 1.2.1 MIDI 입력 방식

MIDI(Musical Instrument Digital Interface)는 디지털 악기 간 통신을 위한 표준 프로토콜로, 피아노 교육 소프트웨어에서 가장 정확한 입력 방식입니다.

**연결 방식**

- **USB-MIDI**: 대부분의 디지털 피아노/키보드가 USB Type-B 단자를 통해 직접 연결 지원
- **MIDI-to-USB 어댑터**: 구형 5핀 DIN MIDI 단자를 USB로 변환
- **Bluetooth MIDI (BLE MIDI)**: iOS/Android에서 무선 연결 지원, Windows는 드라이버 필요

**신호 처리 흐름**

1. 건반 누름 → 디지털 피아노 내부에서 MIDI NoteOn 메시지 생성
2. 메시지 구성: `[상태 바이트(0x90) | 채널] [노트 번호 0-127] [벨로시티 0-127]`
3. USB/BLE를 통해 PC로 전송 (지연: **1ms 이하**)
4. Web MIDI API(브라우저) 또는 네이티브 MIDI 라이브러리로 수신
5. NoteOn/NoteOff 이벤트에서 음표 번호, 세기, 타임스탬프 추출

**MIDI 방식의 장단점**

| 장점 | 단점 |
|---|---|
| 지연 거의 없음 (< 1ms) | MIDI 키보드/피아노 필요 |
| 코드(화음) 인식 완벽 | 어쿠스틱 피아노 사용 불가 |
| 벨로시티(강약) 정보 포함 | — |
| 소음 영향 없음 | — |

---

#### 1.2.2 마이크 오디오 입력 방식

어쿠스틱 피아노나 MIDI 키보드가 없는 사용자를 위한 대안 방식입니다. PC/모바일의 내장 마이크 또는 외부 마이크로 연주 소리를 캡처하여 음정을 인식합니다.

**신호 처리 흐름**

1. 마이크 → Web Audio API `getUserMedia()`로 오디오 스트림 획득
2. `AudioContext` 생성 → `AnalyserNode` 연결
3. 실시간 파형 데이터(`Float32Array`) 수집 (샘플링: 44,100Hz)
4. FFT(Fast Fourier Transform) 또는 자기상관(Autocorrelation) 알고리즘으로 주파수 분석
5. 주파수 → MIDI 노트 번호로 변환

```
MIDI 번호 = 69 + 12 × log₂(Hz / 440)
```

6. 노트 판정 및 타임스탬프 기록

**피치 검출 알고리즘 비교**

| 알고리즘 | 정확도 | 특징 | 적용 예 |
|---|---|---|---|
| FFT + Peak Detection | 단음: 높음 / 화음: 중간 | 스펙트럼 분석, 구현 간단 | 대부분의 튜너 앱 |
| YIN | 단음: 매우 높음 | 자기상관 기반, 낮은 지연 | Guitar/Vocal 튜너 |
| MPM (McLeod Pitch Method) | 단음: 높음 | FFT 결합, 빠른 처리 | 단음 피아노 앱 |
| Spotify Basic Pitch (ML) | 폴리포닉: 높음 | 딥러닝, 화음 인식 가능 | 오디오→MIDI 변환 |

**마이크 방식의 장단점**

| 장점 | 단점 |
|---|---|
| 추가 장비 없이 사용 가능 | 배경 소음에 취약 |
| 어쿠스틱 피아노 지원 | 화음 인식 부정확 |
| — | 지연 약 20~50ms |
| — | 피아노 저음역대 인식 어려움 |

### 1.3 경쟁 제품 핵심 기능 분석

#### 1.3.1 악보 표시 방식

기존 제품들은 크게 두 가지 악보 표시 방식을 사용합니다.

- **폴링 노트(Falling Notes) 방식** — Synthesia 대표: 기타 히어로처럼 화면 위에서 아래로 떨어지는 컬러 블록이 피아노 건반에 닿을 때 누르면 됩니다. 악보를 못 읽는 초보자에게 진입장벽이 낮습니다.
- **스크롤 악보(Sheet Music) 방식** — Simply Piano, Piano Marvel 대표: 실제 오선보가 왼쪽에서 오른쪽으로 스크롤되며 현재 연주해야 할 음표가 강조됩니다. 음악적 리터러시 향상에 유리합니다.
- **하이브리드 방식** — Flowkey: 악보와 함께 가상 피아노 건반에서도 눌러야 할 키를 색상으로 표시합니다.

#### 1.3.2 연주 진행 제어 방식

모든 주요 제품은 **'대기 모드(Wait Mode)'** 를 핵심 기능으로 제공합니다.

- **Wait Mode**: 사용자가 올바른 음표를 누를 때까지 반주/배경음악이 멈추고 대기합니다. 틀린 음을 눌러도 진행되지 않습니다. 초보자의 근육 기억 형성에 최적입니다.
- **자동 진행(Auto) Mode**: 템포에 맞춰 악보가 계속 진행됩니다. 실제 연주 환경과 동일하며 타이밍 훈련에 유리합니다.
- **구간 반복(Loop)**: 어려운 구간을 지정하여 집중 반복 연습합니다. 속도 조절(50~150%) 기능과 함께 제공됩니다.
- **손 분리(Hand Split)**: 왼손 파트와 오른손 파트를 분리하여 연습하고, 소프트웨어가 나머지 파트를 자동 연주합니다.

#### 1.3.3 피드백 시스템

- **음표 정확도**: 올바른 음표는 초록색, 틀린 음표는 빨간색으로 표시
- **타이밍 피드백**: 노트 히트 타이밍의 오차를 ms 단위로 측정하여 표시
- **점수 시스템**: 정확도, 타이밍, 연속 성공률을 종합한 점수 (100점 만점)
- **시각적 건반 하이라이트**: 가상 피아노에서 눌러야 할 건반을 미리 색상으로 표시

## 2. 제품 방향성 및 핵심 차별점

### 2.1 제품 비전

본 프로그램은 Synthesia의 직관적인 비주얼과 Piano Marvel의 체계적인 커리큘럼, Flowkey의 깔끔한 UX를 참고하되, 다음의 차별점을 통해 시장에서 독자적인 위치를 확보합니다.

- **국내 사용자 최적화**: 한국어 UI, 국내 인기 곡 악보 포함 (K-POP, 클래식 한국어 해설)
- **이중 뷰 시스템**: 폴링 노트 뷰와 오선보 뷰를 동시에 표시하거나 전환 가능
- **정밀 마이크 인식**: YIN + MPM 앙상블 알고리즘으로 마이크 정확도 극대화
- **적응형 커리큘럼**: 연주 데이터 분석 → AI 기반 다음 연습 추천
- **모바일 확장성**: PC 설계 단계부터 반응형 구조로 앱 전환이 용이하게 설계

### 2.2 대상 사용자 (Persona)

| 구분 | 페르소나 A — 성인 입문자 | 페르소나 B — 복습/중급자 |
|---|---|---|
| 나이 | 20~40대 | 10~30대, 과거 피아노 경험 있음 |
| 목표 | 좋아하는 곡 연주, 취미 개발 | 실력 향상, 악보 읽기 강화 |
| 장비 | 저가 MIDI 키보드 또는 어쿠스틱 | 중급 디지털 피아노 (MIDI 지원) |
| 선호 기능 | Wait Mode, 폴링 노트, 쉬운 곡 | 악보 뷰, 상세 분석, 어려운 곡 |

## 3. 기술 아키텍처 설계

### 3.1 전체 시스템 구조

본 시스템은 5개 레이어로 구성됩니다. 각 레이어는 독립적으로 동작하며, 공통 이벤트 버스를 통해 통신합니다.

                            ▼
┌─────────────────────────────────────────────────────────────┐
│                      Layer 2: 신호 처리 엔진                  │
│     [MIDI 파서]         [FFT/YIN 피치 검출]    [이벤트 정규화]   │
│  NoteOn/Off, Velocity   폴리포닉 인식          공통 NoteEvent  │

                            ▼
┌─────────────────────────────────────────────────────────────┐
│                     Layer 3: 비교 / 판정 엔진                 │
│   [음표 비교기]         [리듬 분석기]         [진행 제어]        │

│                      Layer 4: UX / 피드백 레이어              │

### 3.3 MIDI 신호 처리 상세 설계

MIDI 입력은 가장 정확한 방식이므로 최우선 입력 경로로 설계합니다.

**Web MIDI API 초기화 흐름**

1. `navigator.requestMIDIAccess()` 호출 → 사용자 권한 요청
2. `MIDIAccess` 객체에서 `inputs` 목록 열거 → 사용자에게 기기 선택 UI 제공
3. 선택된 `MIDIInput`에 `onmidimessage` 이벤트 리스너 등록
4. 이벤트 발생 시 `event.data(Uint8Array)`에서 `[상태바이트, 노트번호, 벨로시티]` 파싱
5. NoteOn(`0x90`, velocity > 0): 음표 시작 이벤트 emit
6. NoteOff(`0x80`) 또는 NoteOn(velocity=0): 음표 종료 이벤트 emit

**MIDI 노트 이벤트 데이터 구조 (NoteEvent)**

```typescript
interface NoteEvent {
  noteNumber: number;          // 0~127 (가운데 도 = 60)
  velocity: number;            // 0~127 (음의 세기)
  timestamp: DOMHighResTimeStamp; // ms 단위, 고해상도
  type: 'on' | 'off';
  channel: number;             // 0~15
}
```

### 3.4 오디오(마이크) 피치 검출 상세 설계

마이크 입력은 MIDI가 없는 사용자를 위한 대안 경로입니다. 초보자 연주 시 단음(멜로디) 위주로 사용하도록 가이드합니다.

**피치 검출 파이프라인**

1. Web Audio API `AudioContext` 생성 (sampleRate: 44100Hz)
2. `getUserMedia({ audio: true })` → `MediaStream` 획득
3. `createMediaStreamSource()` → `AnalyserNode` 연결
4. `fftSize: 4096` 설정 (주파수 해상도 10.77Hz)
5. `getFloatTimeDomainData()`로 시간축 파형 데이터 추출
6. YIN 알고리즘으로 기본 주파수(F0) 추정
7. F0 → MIDI 번호 변환 → 노이즈 게이팅 (RMS < -40dBFS 시 무시)
8. Note-on/off 상태 머신으로 연속 감지 중 노트 경계 판정

**화음 인식의 한계와 대응**

마이크 방식의 최대 단점은 폴리포닉(화음) 인식의 부정확성입니다. 다음과 같이 대응합니다.

- 마이크 모드에서는 '단음 연습' 콘텐츠만 제공하도록 커리큘럼 필터링
- 화음이 필요한 악보는 MIDI 연결 권장 안내 팝업 표시
- 고급 마이크 인식: Spotify Basic Pitch(ML 기반) 통합 검토 (v2.0 로드맵)

## 4. 비교 / 판정 엔진 설계

### 4.1 음표 정확도 판정

연주 입력(NoteEvent)과 악보 데이터(ScoreNote)를 실시간으로 비교합니다.

| 판정 결과 | 조건 | UI 표현 |
|---|---|---|
| **Perfect** (완벽) | 올바른 음 + 타이밍 오차 ±30ms | 초록색 하이라이트 + 파티클 효과 |
| **Good** (양호) | 올바른 음 + 타이밍 오차 ±100ms | 연두색 하이라이트 |
| **Late / Early** | 올바른 음 + 타이밍 오차 > 100ms | 노란색 + 'Late' / 'Early' 텍스트 |
| **Miss** (오음) | 틀린 음 입력 | 빨간색 + 올바른 음표 강조 |
| **Skip** (누락) | 다음 음으로 넘어갈 때까지 미입력 | 회색 처리 + 누락 카운트 증가 |

### 4.2 점수 산출 공식

```
점수 = (음정 정확도 × 0.5) + (타이밍 정확도 × 0.3) + (완주율 × 0.2)
```

- **음정 정확도**: `(Perfect + Good 음표 수) / 전체 음표 수 × 100`
- **타이밍 정확도**: 타이밍 오차 평균을 기반으로 0~100 선형 환산
- **완주율**: 전체 곡 중 포기 없이 완주한 비율 (구간 반복 포함)

### 4.3 진행 제어 로직 설계

#### 4.3.1 Wait Mode (대기 모드)

1. 현재 악보 위치의 '목표 음표 집합' 결정 (단음 또는 화음)
2. 사용자 입력 대기 (타임아웃 없음, 무한 대기)
3. 입력된 음표가 목표 집합과 일치하면 → 다음 음표로 진행 (반주 재개)
4. 불일치 시 → 빨간색 피드백 표시 후 계속 대기 (반주 정지 유지)
5. 힌트 버튼: 눌러야 할 건반을 건반 뷰에서 깜빡임으로 안내

#### 4.3.2 Play Mode (자동 진행)

1. BPM 설정에 따라 반주 트랙 시작
2. 음표 이벤트와 연주 입력을 동시에 타임라인에서 비교
3. 진행은 BPM 기준으로 자동, 미스 시에도 계속 진행
4. 구간 종료 후 결과 리포트 표시 (음표별 판정 결과 시각화)

## 5. 사용자 인터페이스(UX) 설계

### 5.1 메인 화면 레이아웃

메인 연습 화면은 3개의 주요 영역으로 구성됩니다.

```
┌─────────────────────────────────────────────────────────────┐
│  상단 (20%)  │ 제목 / BPM / 모드 선택 / 점수 실시간 표시        │
├─────────────────────────────────────────────────────────────┤
│              │                                               │
│  중앙 (60%)  │   악보 렌더링 영역                              │
│              │   (폴링 노트 ↔ 오선보 전환 가능)                │
│              │                                               │
├─────────────────────────────────────────────────────────────┤
│  하단 (20%)  │ 88건반 가상 피아노 / 손 분리 / 재생 컨트롤        │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 화면별 구성

| 화면 | 주요 요소 |
|---|---|
| 홈 / 대시보드 | 학습 통계 요약, 오늘의 추천 연습, 최근 연주 이력, 업적 현황 |
| 곡 라이브러리 | 장르/난이도/아티스트 필터, 악보 미리 듣기, 즐겨찾기 |
| 연습 화면 | 악보 뷰 + 가상 건반 + 실시간 판정 피드백 + 컨트롤 바 |
| 결과 리포트 | 음표별 판정 결과 리플레이, 점수, 개선 포인트 AI 코멘트 |
| 커리큘럼 | 레벨별 단계 진도 표시, 잠금/해제 구조, 이론 레슨 영상 |
| 설정 | 입력 기기 선택, 마이크 보정, 손 크기/난이도 설정, 알림 |

### 5.3 게이미피케이션 시스템

사용자 동기 유발을 위해 다음 게임화 요소를 적용합니다.

- **XP 포인트**: 연습 시간, 정확도, 새로운 곡 클리어에 따라 경험치 부여
- **레벨 시스템**: 레벨 1(입문) ~ 레벨 10(전문가) 단계별 칭호
- **연속 연습 스트릭**: 매일 연습 시 스트릭 카운터 증가, 7일/30일 달성 배지
- **곡별 별점**: 3회 이상 연주 후 최고 점수 기준 별 1~3개 부여
- **주간 챌린지**: 특정 곡을 정해진 기간 내 클리어하는 시간 제한 챌린지
- **업적 시스템**: '첫 100점', '100곡 완주', '1000시간 연습' 등 마일스톤 배지

## 6. 데이터 모델 설계

### 6.1 핵심 데이터 엔티티

#### User

```typescript
interface User {
  userId: string;
  nickname: string;
  level: number;
  totalXP: number;
  streak: number;             // 연속 연습일
  lastPracticeDate: string;
  settings: {
    inputMode: 'midi' | 'mic' | 'virtual';
    bpm: number;
    viewMode: 'falling' | 'sheet' | 'hybrid';
    handSplit: 'both' | 'left' | 'right';
  };
}
```

#### Song (악보)

```typescript
interface Song {
  songId: string;
  title: string;
  artist: string;
  genre: string;
  difficulty: number;         // 1~10
  durationMs: number;
  bpm: number;
  keySignature: string;
  timeSignature: string;
  midiData: string;           // Base64 encoded MIDI binary
  musicXML: string;
  audioPreview: string;       // URL
}
```

#### PracticeSession (연습 기록)

```typescript
interface PracticeSession {
  sessionId: string;
  userId: string;
  songId: string;
  startedAt: string;
  endedAt: string;
  mode: 'wait' | 'play';
  totalScore: number;
  pitchAccuracy: number;
  timingAccuracy: number;
  completionRate: number;
  noteResults: NoteResult[];
}
```

#### NoteResult (음표별 판정)

```typescript
interface NoteResult {
  noteIndex: number;          // 악보상 위치
  noteNumber: number;         // MIDI 번호
  expectedTimeMs: number;
  actualTimeMs: number;
  timingDeltaMs: number;
  verdict: 'perfect' | 'good' | 'late' | 'early' | 'miss' | 'skip';
}
```

### 6.2 로컬 저장소 구조 (IndexedDB)

초기 PC 버전은 별도 서버 없이 로컬 IndexedDB에 모든 학습 데이터를 저장합니다. 향후 앱 출시 시 클라우드 동기화(서버 API)를 추가합니다.

| 테이블 | 내용 |
|---|---|
| `songs` | 악보 메타데이터 및 MIDI/MusicXML 데이터 |
| `sessions` | 연습 기록 (noteResults는 JSON 직렬화 저장) |
| `achievements` | 획득 배지 목록 |
| `userProfile` | 사용자 설정, 레벨, XP |

## 8. 기술적 리스크 및 대응 방안

| 리스크 | 심각도 | 대응 방안 |
|---|---|---|
| Web MIDI API가 Safari에서 미지원 | 🔴 높음 | Electron 앱으로 우회. 웹 버전은 Chrome/Edge 권장 명시 |
| 마이크 화음 인식 정확도 부족 | 🟡 중간 | 마이크 모드는 단음 전용 콘텐츠 제한. v2.0에서 ML 도입 |
| MusicXML 복잡한 악보 렌더링 오류 | 🟡 중간 | VexFlow 업스트림 이슈 추적. 자체 악보 편집기 도입 검토 |
| 저가 MIDI 키보드의 높은 지연 | 🟢 낮음 | 타이밍 허용 오차 설정값 기기별 교정 기능 제공 |
| 악보 저작권 문제 | 🔴 높음 | 퍼블릭 도메인 곡 우선. 저작권 곡은 라이선스 계약 후 추가 |
